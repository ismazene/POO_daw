Map + Iterator en Java

// chuleta rápida · examen práctico
Crear un Map 
// HashMap — el más común, sin orden
Map<String, Integer> mapa = new HashMap<>();

// TreeMap — ordenado por clave (A-Z)
Map<String, Integer> mapa = new TreeMap<>();

// LinkedHashMap — mantiene orden de inserción
Map<String, Integer> mapa = new LinkedHashMap<>();
Import: import java.util.HashMap;  /  import java.util.*;
Añadir y modificar 
// put(clave, valor) — añade o sobreescribe
mapa.put("Ana", 25);
mapa.put("Luis", 30);
mapa.put("Ana", 26);  // sobreescribe el 25

// putIfAbsent — solo añade si la clave NO existe
mapa.putIfAbsent("Ana", 99);  // no hace nada (ya existe)
mapa.putIfAbsent("Eva", 22);   // sí añade
Buscar y obtener 
// get(clave) — devuelve el valor o null
Integer edad = mapa.get("Ana");  // → 26

// getOrDefault — valor por defecto si no existe
int e = mapa.getOrDefault("Pep", 0);  // → 0

// containsKey — ¿existe la clave?
if (mapa.containsKey("Luis")) { ... }

// containsValue — ¿existe el valor?
if (mapa.containsValue(30)) { ... }
Eliminar 
// remove(clave) — elimina por clave
mapa.remove("Luis");

// remove(clave, valor) — solo elimina si coinciden ambos
mapa.remove("Ana", 26);   // elimina solo si Ana → 26

// clear() — vacía el mapa entero
mapa.clear();

// isEmpty() — ¿está vacío?
boolean vacio = mapa.isEmpty();

// size() — número de entradas
int n = mapa.size();
Recorrer el mapa — 3 formas 
// 1. entrySet() — clave Y valor a la vez (la más usada)
for (Map.Entry<String, Integer> entry : mapa.entrySet()) {
    System.out.println(entry.getKey() + " → " + entry.getValue());
}

// 2. keySet() — solo claves
for (String clave : mapa.keySet()) {
    System.out.println(clave);
}

// 3. values() — solo valores
for (Integer valor : mapa.values()) {
    System.out.println(valor);
}

// 4. forEach con lambda (forma moderna)
mapa.forEach((clave, valor) ->
    System.out.println(clave + " → " + valor));
Eliminar entradas mientras recorres — con Iterator 
// NO puedes usar for-each + remove() → ConcurrentModificationException
// SOLUCIÓN: Iterator sobre entrySet()

Iterator<Map.Entry<String, Integer>> it = mapa.entrySet().iterator();
while (it.hasNext()) {
    Map.Entry<String, Integer> entry = it.next();
    if (entry.getValue() < 25) {    // condición para eliminar
        it.remove();               // SIEMPRE it.remove(), nunca mapa.remove()
    }
}
Regla de oro: para eliminar mientras recorres → Iterator + it.remove(). Igual que con ArrayList.
Ejemplo completo — gestión de notas de alumnos 
import java.util.*;

public class EjemploMapa {
    public static void main(String[] args) {

        Map<String, Double> notas = new HashMap<>();

        // Añadir
        notas.put("Ana", 8.5);
        notas.put("Luis", 4.0);
        notas.put("Eva", 9.0);
        notas.put("Pep", 3.5);

        // Buscar
        System.out.println("Nota de Ana: " + notas.get("Ana"));  // 8.5

        // Modificar nota de Luis
        notas.put("Luis", 5.0);

        // Eliminar los suspensos (nota < 5) con Iterator
        Iterator<Map.Entry<String, Double>> it = notas.entrySet().iterator();
        while (it.hasNext()) {
            Map.Entry<String, Double> entry = it.next();
            if (entry.getValue() < 5.0) {
                it.remove();
            }
        }

        // Imprimir los que quedan
        for (Map.Entry<String, Double> e : notas.entrySet()) {
            System.out.println(e.getKey() + ": " + e.getValue());
        }
        // → Ana: 8.5   Luis: 5.0   Eva: 9.0   (Pep eliminado)
    }
}
Resumen de métodos — referencia rápida 
método                      qué hace
──────────────────────────────────────────────────────────────
mapa.put(k, v)              añade / sobreescribe
mapa.putIfAbsent(k, v)      añade solo si la clave no existe
mapa.get(k)                 devuelve valor o null
mapa.getOrDefault(k, def)   devuelve valor o 'def' si no existe
mapa.containsKey(k)         true si la clave existe
mapa.containsValue(v)       true si el valor existe
mapa.remove(k)              elimina por clave
mapa.remove(k, v)           elimina solo si clave+valor coinciden
mapa.size()                 número de pares clave-valor
mapa.isEmpty()              true si está vacío
mapa.clear()                vacía el mapa
mapa.keySet()               Set con todas las claves
mapa.values()               Collection con todos los valores
mapa.entrySet()             Set de Map.Entry (clave + valor)
Iterator en listas // ArrayList, LinkedList, cualquier Collection
Métodos del Iterator 
método          qué hace
─────────────────────────────────────────────
it.hasNext()    true si quedan elementos
it.next()       devuelve el siguiente y avanza
it.remove()     elimina el último devuelto por next()
Orden obligatorio: siempre next() ANTES de remove(), si no → IllegalStateException
Recorrer una lista con Iterator 
ArrayList<String> lista = new ArrayList<>();
lista.add("Ana");
lista.add("Luis");
lista.add("Eva");

Iterator<String> it = lista.iterator();

while (it.hasNext()) {
    String elemento = it.next();
    System.out.println(elemento);
}
// → Ana  Luis  Eva
Equivalente al for-each, pero dándonos control total sobre el cursor.
Eliminar mientras recorres 
ArrayList<Integer> nums =
    new ArrayList<>(Arrays.asList(2,4,6,8,10));

Iterator<Integer> it = nums.iterator();
while (it.hasNext()) {
    int n = it.next();
    if (n > 5) {
        it.remove(); // elimina el elemento actual
    }
}
// nums → [2, 4]
Nunca lista.remove() dentro del while → ConcurrentModificationException
For-each vs Iterator 
// FOR-EACH — simple, solo lectura
for (String s : lista) {
    System.out.println(s);  // OK
    lista.remove(s);        // ❌ EXCEPCIÓN
}

// ITERATOR — lectura + eliminación segura
Iterator<String> it = lista.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);  // OK
    it.remove();            // ✓ seguro
}
ListIterator — versión extendida (puede ir hacia atrás y modificar) 
ListIterator<String> lit = lista.listIterator();

// Métodos extra que tiene ListIterator:
método              qué hace
──────────────────────────────────────────────────────────
lit.hasNext()       true si hay siguiente
lit.next()          devuelve el siguiente y avanza →
lit.hasPrevious()   true si hay anterior
lit.previous()      devuelve el anterior y retrocede ←
lit.nextIndex()     índice del siguiente elemento
lit.previousIndex() índice del anterior elemento
lit.remove()        elimina el último devuelto
lit.set(e)          reemplaza el último devuelto por e
lit.add(e)          inserta e antes de la posición actual

// Ejemplo: recorrer al revés
ListIterator<String> lit = lista.listIterator(lista.size());
while (lit.hasPrevious()) {
    System.out.println(lit.previous());
}

// Ejemplo: reemplazar elementos mientras recorres
ListIterator<String> lit2 = lista.listIterator();
while (lit2.hasNext()) {
    String s = lit2.next();
    lit2.set(s.toUpperCase());  // convierte cada elemento a mayúsculas
}
ListIterator solo funciona con listas (ArrayList, LinkedList). El Iterator básico funciona con cualquier Collection (Set, Map.entrySet…)
Cuándo usar cada uno — resumen 
situación                                   solución
──────────────────────────────────────────────────────────────────
solo leer la lista                          for-each  (más simple)
eliminar elementos mientras recorres        Iterator  + it.remove()
recorrer al revés                           ListIterator.hasPrevious()
modificar elementos mientras recorres       ListIterator.set()
eliminar de un Map mientras recorres        Iterator sobre entrySet()
insertar en medio mientras recorres         ListIterator.add()
IDs y matrículas aleatorias // Random, UUID, String.format
Random — números aleatorios 
import java.util.Random;

Random rnd = new Random();

// entero entre 0 y 99 (0 incluido, 100 excluido)
int n = rnd.nextInt(100);

// entero entre min y max (ambos incluidos)
int min = 1000, max = 9999;
int n = rnd.nextInt(max - min + 1) + min;

// también hay: nextDouble(), nextBoolean(), nextLong()
double d = rnd.nextDouble();  // 0.0 a 1.0
Atajo sin crear objeto: Math.random() devuelve double entre 0.0 y 1.0
→ (int)(Math.random() * 100) da entero 0–99
NIA / matrícula numérica 
Random rnd = new Random();

// NIA de 5 dígitos: 10000 – 99999
int nia = rnd.nextInt(90000) + 10000;
// → ej: 47382

// Con ceros a la izquierda fijos (ej: 00042)
String niaPad = String.format("%05d", rnd.nextInt(100000));
// → ej: "00873"

// Matrícula tipo "ALU-00042"
String matricula = "ALU-" + String.format("%05d", rnd.nextInt(100000));
// → ej: "ALU-00873"
UUID — ID único universal 
import java.util.UUID;

// genera un ID único garantizado (36 caracteres)
String id = UUID.randomUUID().toString();
// → "a3f2c1d0-4e5b-11ee-be56-0242ac120002"

// versión corta: solo los primeros 8 caracteres
String idCorto = id.substring(0, 8).toUpperCase();
// → "A3F2C1D0"
UUID es prácticamente imposible que se repita. Ideal para IDs de base de datos.
Matrícula con letras aleatorias 
Random rnd = new Random();

// letra aleatoria A-Z
char letra = (char) ('A' + rnd.nextInt(26));
// → ej: 'K'

// matrícula tipo "AB-4721" (2 letras + 4 números)
char l1 = (char) ('A' + rnd.nextInt(26));
char l2 = (char) ('A' + rnd.nextInt(26));
int  num = rnd.nextInt(9000) + 1000;  // 1000–9999
String mat = "" + l1 + l2 + "-" + num;
// → ej: "KR-4721"
Ejemplo completo — generar lista de alumnos con NIA único 
import java.util.*;

public class GeneradorAlumnos {
    public static String generarNIA() {
        Random rnd = new Random();
        return "NIA-" + String.format("%05d", rnd.nextInt(100000));
    }

    public static void main(String[] args) {
        // Map que garantiza NIAs únicos: NIA → nombre
        Map<String, String> alumnos = new HashMap<>();
        String[] nombres = {"Ana", "Luis", "Eva", "Pep"};

        for (String nombre : nombres) {
            String nia;
            do {
                nia = generarNIA();           // genera uno nuevo
            } while (alumnos.containsKey(nia)); // repite si ya existe
            alumnos.put(nia, nombre);
        }

        // imprimir resultado
        for (Map.Entry<String, String> e : alumnos.entrySet()) {
            System.out.println(e.getKey() + " → " + e.getValue());
        }
        // → NIA-04821 → Ana
        // → NIA-72914 → Luis  ...
    }
}
Truco: el bucle do-while con containsKey garantiza que no se repita ningún NIA en el mapa.
